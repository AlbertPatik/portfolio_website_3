Webové stránky

Cílem tohoto miniprojektu bylo vytvořit si webové portfolio k prezentaci našich projektů v rámci předmětu ZPC.

Hugo nebo HTML?

Během výuky jsme byli vedeni k použití nástroje Hugo, který při správném nastavení umožňuje relativně jednoduše tvořit webové stránky z markdownových souborů. Jelikož jsem však už měl nějaké zkušenosti s vývojem webových stránek, rozhodl jsem se použít mnohem robustnější sadu technologií – HTML, CSS a TailwindCSS – která není o moc složitější.

Jak to funguje?

Jelikož jsme stránku měli hostovat pomocí služby Github Pages, bylo potřeba, aby stránka byla statická – tedy není vyžadován server a všechna data se posílají přímo klientovi do jeho prohlížeče.

HTML definuje strukturu a obsah stránky. Snažil jsem se vyhnout používání JavaScriptu a veškerou logiku aplikace řešit pouze pomocí vestavěných funkcí HTML a CSS. Kromě použití TailwindCSS jsem žádný další JavaScript nepotřeboval.

CSS pak obsahu stránky dává vzhled. Umožňuje měnit barvy, rozložení prvků a mnohé další. I když se z pohledu vývoje webových stránek jedná o poměrně staré technologie, je možné v nich vytvořit velice profesionální stránky bez použití dalších nástrojů.

I přesto, že bych si vystačil s HTML a CSS, rozhodl jsem se použít i TailwindCSS, což je nadstavba nad CSS, která umožňuje velice rychle tvořit nádherně vypadající stránky. S čistým CSS by tvorba stejné stránky trvala déle.

A co dál?

Vpodstatě už stačí jenom správně nastavit automatický deployment na GitHub Pages a portfolio je hotové. Zvolený postup je jednoducý, robustní a veřejně dostupné jazykové modely s ním dokážou velice spolehlivě pomoct, což je obrovská výhoda při práci na jekémkoliv projektu.
